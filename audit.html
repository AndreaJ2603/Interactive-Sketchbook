Interface Audit — Micro Project

Visibility of System Status

I have rechecked my page several times and every time I go back I can see that I could improve.
The page communicates to the user where they are specifically because it stated clearly that to start you must to click anywhere. The page is also interactive because the user would need to use the mouse to interact with the game. The weakness when it comes to visibility that I can identify is that the random clicking could create an issue for issuers.

Affordances & Signifiers

I think for affordability the only action available for the user is to click and get mad! The action is informed to users by words. I used a bright color green to encourage the user to read it. It is incredibly clickable because that is the idea of the interaction click to earn points. But there is not a specific visual signifier to guide the user to click at a specific place to start the game.

Navigation & Mapping

The system is stale and unpredictable for mapping because the colors change too quickly.The layout definitely reflects the system’s structure because it lacks flexibility and user appeal, being overly simple and lacking hierarchical structure.
The only predictability that the user would experience is that the shapes will always be moving in one direction at the same continuous speed.
I think because of these faults the user might hesitate to play this game.

Constraints & Assumptions

I think the page assumes that the user will read the instruction to “click any key.” However, a typical user’s immediate response would likely be to click the mouse, since the entire game revolves around mouse clicks. What isn’t clear is that the game doesn’t specify which key to press, for example.
It says click any key…it is open…I could specify to click “L” or “B”—and using the specific command clearer. Additionally, the game restricts the user from restarting at any point and doesn’t allow going back to previous steps

Heuristic Violations

I have identified that User Control and Freedom is the most apparent violation of heuristics principles. Because the user would not have a choice to go back, stop the game or start over.
The other major Heuristics violation I found in my project was consistency and standards. Simply because the actions are not reversible. The user must finish the game in order to start over.
I would propose to redesign the structure of the page by creating a button where the user can click to stop or pause the game and also go back at any time during the game. These changes would improve both control and freedom as well as consistency and standards.

Structural Revision Proposal

I would propose structural changes to have a clear visual hierarchy by adding a button to click to start the game instead of a verbal command to click any key.
And the navigation I would change the layout of the page by adding a trail in which the shape would follow. Additionally state visibility like the counting down clock and score would be positioned in one side of the screen instead of one in each side. To improve interaction clarity I would change the speed in which the colors change. I would make it click with rhythm instead of chaotic and random.
