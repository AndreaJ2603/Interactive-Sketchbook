Draft Interface System Statement

Interface System Prototype #3: Full Rough Draft

I have focused on redesigning a simple, game interface and I made sure that I have prioritized clarity and consistency. I have implemented the usability by adding clear signals and wording to instruct users. The user is introduced with a clear entry to play the game by clicking on designated tabs.

The core interaction is straightforward—players must tap or click an octagon when it changes color— I have also implemented clear language to tap on cell phones and on the desktop I made sure to state to click space bar.

I have also implemented an interface with a flat navigation menu to avoid confusion and distraction for the user.

The octagon and the flower is the central focus of the screen, I used sized appropriately to draw immediate attention

Norman’s principles are reflected in the use of clear affordances and feedback. The octagon’s color change acts as an obvious signal for interaction signaling for the user to click to score points. This aligns with normans also where the user actions give immediate results.

Jakob Nielsen’s usability heuristics are also applied throughout the design The design follows consistency and standards by using familiar interaction, like tapping. I have also tried to implement a minimalist design that aligns with Nilsen’s principle of aesthetics to make sure that only the essential elements are displayed on screen

From the perspective of Shneiderman's Eight Golden Rules I have implemented consistency throughout the game. Users have an informative feedback displayed on screen during playing games. This simplistic design with simple navigation and instruction decreases the user’s memory overload.

I think I can still make changes to improve the aesthetics of this design. I aim to create a game that not only improves the user experience but also enhances affordance
